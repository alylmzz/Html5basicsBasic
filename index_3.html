Section Three

Block Level Elements

block level elements takes 100 percent width of the page. (stretches out to the left and right as far as it can).So it basicially goes across the whole page and next thing you put on the page will be on next line, For example a paragraph ( <p></<p> ) is a block level element, so by default if you try to put another one next to it, even if they have 3 or 4 characters inside, and even if the codes were written side by side, that 3 character paragraph will push the other one, down by one line . Block level elements can be styled through CSS so that we can get the look we want on our page.

Inline Level Elements

An inline element does not span accross the page. It only takes up that space it stays therefore next thing you put on page will just go to the right side of it . Example , an image is an inline level element, so if we put two of it at same line of code, in browser they would stay side by side. So proper and desired positioning and styling can be achived through CSS

Id's

id="" is a uniqe identifier that can only be used once. every id="" must be uniqe to where it used. Can not be used elsewhere. So when we use CSS or JavaScript to manipulate, we will know that we are only changing a certain thing. Example;
<ul id="main-nav" >
    <li><a href="#">Home Page</a></li>
</ul>
. So whenever we want to manipulate this list in CSS ot JavaScript, we will use id="main-nav".

Classes

Classes are another identification method for HTML. Unlike id="", classes can be used multiple items. This way if we want to manipulate multiple things at once we wont have to change them one by one.

Entities

Entities are short character combinations for different types of characters like Copyright © or even codes themselves.
<code>
<title>Entities</title >
</code>

The Division Tag

The <div> tag is nothing more than a container unit that encapsulates other page elements , and divides the HTML document into sections. Web developers use <div> elements to group together HTML elements and apply CSS styles to many elements at once. For instance, by wrapping a set of paragraph elements into a <div> element, the developer can take advantage of CSS styles and apply a font to all paragraphs at once by applying a font style to the <div> tag instead of coding the same style for each paragraph element.

Semantics

Semantics are used for giving meaning to mainly used divisions. For example rather then wrapping main header to a <div> tag we can use <header> tag to make it certain in the beginning. Here are some examples of semantics. header>, <nav>, <footer>, <aside> Using semantics does not mean we can not use division. As we said before semantics are generally used for main divisions like headers or navigation. But for example in a main division we may need other divisions as well. Like in a galerry section we can create black & whitw and colorful photos divisions. Semantics and division used together for developers to get roadmap for themselves.
